# LangChain 简介

LangChain 是一个用于开发由语言模型驱动的应用程序的框架。它提供了一套工具、组件和接口，使开发者能够轻松构建复杂的 LLM 应用。

## 核心概念

### 1. Chains（链）
Chains 允许我们将多个组件组合在一起，创建一个单一的应用程序。例如，我们可以创建一个链，它接受用户输入，使用 PromptTemplate 格式化它，然后将格式化的响应传递给 LLM。

### 2. Agents（代理）
Agents 是 LangChain 中最强大的功能之一。代理可以使用 LLM 来决定采取什么行动以及以什么顺序采取行动。行动可以是使用工具并观察其输出，也可以是返回给用户。

### 3. Memory（记忆）
默认情况下，Chains 和 Agents 是无状态的。但在许多应用中，记住之前的交互是非常重要的。Memory 组件可以帮助我们在多次调用之间保持状态。

### 4. Retrieval（检索）
Retrieval 是指根据用户的问题获取相关外部数据的过程。这对于问答系统特别有用，因为它允许我们基于特定的数据源生成答案，而不仅仅是依赖模型的训练数据。

## RAG（检索增强生成）

RAG 是 Retrieval-Augmented Generation 的缩写，它将信息检索与文本生成结合起来。基本流程如下：

1. **索引阶段**：将文档分割成小块，转换为向量，存储在向量数据库中
2. **检索阶段**：将用户查询转换为向量，在向量数据库中搜索最相关的文档
3. **生成阶段**：将检索到的文档和用户查询一起发送给 LLM，生成回答

## 主要优势

- **准确性**：基于真实数据生成回答，减少幻觉
- **可追溯性**：可以提供信息来源
- **时效性**：可以检索最新的信息
- **定制化**：可以针对特定领域进行优化

## 应用场景

- 智能客服系统
- 企业知识库问答
- 文档分析和总结
- 研究助手
- 教育辅导系统
